# 用 AI 编写文档

AI 可以帮助你起草页面、补齐示例、生成中英文版本和检查文档结构，但最终仍建议由维护者审阅。

## 推荐工作流 {#workflow}

可以按这个流程使用 AI：

1. 先写出页面目标和读者对象。
2. 让 AI 生成初稿和目录。
3. 手动补充真实命令、路径和项目约束。
4. 让 AI 检查是否缺少前置条件、失败场景和下一步。
5. 本地运行站点，人工检查页面阅读体验。

## 示例提示词 {#prompt}

下面是一个适合生成文档初稿的提示词：

```text
你正在为一个基于 Vue、Vite、mdui 2 和 Markdown 的开源文档系统写文档。
请为“部署发布”页面生成中文初稿。
要求包含构建命令、本地预览、GitHub Pages 示例和发布前检查清单。
文风要清晰、偏工程实践，不要营销化。
```

## 翻译页面 {#translation}

你可以让 AI 根据中文页面生成英文版：

```text
请把下面的中文 Markdown 文档翻译成英文。
保留标题锚点、代码块、链接路径和列表结构。
不要改变文件路径或命令。
```

翻译后建议人工检查专业名词、链接路径和语气是否符合项目风格。

## 审查清单 {#review-checklist}

AI 生成内容后，请重点检查：

- 命令是否真实可运行。
- 文件路径是否存在。
- 链接是否指向当前项目。
- 中英文页面标题锚点是否一致。
- 是否出现与项目无关的第三方 API 或不存在的配置项。
